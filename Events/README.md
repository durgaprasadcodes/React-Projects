## ✨ Capabilities (What It Does)

✔ Add tasks through a controlled input
✔ Delete tasks using immutable state updates
✔ Render lists declaratively and safely
✔ Maintain a single source of truth
✔ Clean re-render cycles
✔ Zero side effects

No hacks.
No mutations.
No surprises.

---

## 🧠 Technical Concepts Demonstrated

### 🔹 State Management

* `useState` as a single source of truth
* Immutable updates (`map`, `filter`)
* Explicit state transitions

### 🔹 Rendering Strategy

* Declarative UI
* Stable reconciliation using keys
* No unnecessary re-renders

### 🔹 Event Architecture

* Explicit `onClick` handlers
* Argument-safe callbacks
* Clear separation between UI & logic

This mirrors **real production React patterns**.

## 🎨 Design Philosophy

The UI is intentionally:

* Calm
* Focused
* Distraction-free

Design choices prioritize:

* Readability over decoration
* Structure over animation noise
* Usability over gimmicks

The result is a UI that feels **professional, not playful**.

## 🧩 Architecture Mindset

This project is designed to be **extended**, not rewritten.

It cleanly supports:

* Keyboard interactions (Enter key)
* Persistent storage (`localStorage`)
* Task state transitions (complete / edit)
* Animations via declarative libraries
* Component extraction and reuse

The foundation is solid — growth is natural.
